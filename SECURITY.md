# 安全政策

## 支持的版本

zimaflow 目前处于 v0.1 首次公开发行阶段，安全修复只针对最新发布版本。

| 版本 | 是否支持 |
|------|----------|
| 0.1.x | ✅ |
| < 0.1 | ❌ |

## 报告漏洞

zimaflow 是一套本地运行、无网络、无凭证依赖的工作流脚本与文档集。即便如此，如果你发现潜在安全问题（例如安装脚本或 CLI 存在命令注入、路径穿越、意外写入用户目录之外等风险），请**不要**公开提 issue。

请通过 GitHub 的私密漏洞报告通道提交：

1. 打开本仓库的 **Security** 标签页。
2. 点击 **Report a vulnerability**，按表单填写。
3. 尽量提供复现步骤、受影响文件和你认为的影响范围。

该通道全程私密，只有维护者可见，无需公开任何联系方式。我会在收到后尽快确认，并在修复发布后与你同步。请在修复公开前对细节保密。

## 范围说明

以下不属于安全问题，请走普通 issue：

- 文档措辞、链接错误或表述不清。
- 主链路流程设计上的建议。
- 功能缺失或功能请求（请走 feature request issue）。
