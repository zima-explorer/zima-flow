# 工时估算基准字典

> task-planning Skill 参考此文件进行工时估算。
> 项目可通过 `<docs_dir>/project-workload-overrides.md` 覆盖特定条目。
> 每次复盘时根据实际 vs 估算的偏差校准。

---

## 后端

| 任务类型 | 基准工时 | 说明 |
|---------|---------|------|
| 新增标准 CRUD 接口（单表） | 3h | 含 route + service + 基础校验 |
| 新增复杂接口（跨表/事务） | 5h | 含事务处理、错误补偿 |
| 修改现有接口（加字段/改逻辑） | 1-2h | 视改动范围 |
| 新增中间件/拦截器 | 2h | 鉴权、日志、限流等 |
| 数据库 DDL 变更（加表/加字段） | 1h | 含迁移脚本 |
| 数据库 DDL 变更（改表结构） | 2-3h | 需考虑数据迁移 |
| 外部服务对接（SDK/API） | 4h | 含封装、错误处理、重试 |
| 文件上传/下载（对象存储） | 3h | 含流式处理、大小限制校验 |
| 定时任务/后台 Job | 3h | 含幂等设计 |
| Bug 修复（定位明确） | 0.5-1h | 已知复现路径 |
| Bug 修复（需排查） | 2-4h | 需日志分析或调试 |

## 使用说明

- 工时包含开发+自测时间，不含等待（CR 等待、部署排队等）
- 如果任务明显超出基准范围，在备注中说明原因
- 基准值是中位数，复杂度高的项目应上浮 30-50%
- 每完成一轮开发复盘时，对比实际工时 vs 估算，偏差超过 50% 的条目需要校准
